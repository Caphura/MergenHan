# Resume Composer Session Example

This example shows how `Resume Composer` can turn a messy evidence pack into a stronger, ATS-friendly English resume draft.

## Raw Request

I do not have a clean resume. I can give you LinkedIn screenshots, a Canva-exported PDF, a GitHub repo, and two public articles about a game project. Build me a one-page English resume and do not invent anything.

## Expected Clarification Discipline

- The model should first recognize this as an inspectable evidence pack.
- It should inspect screenshots, exported PDF text, repo documentation, and public articles before opening a clarification round.
- If questions are still needed, it should ask only one short round of high-impact questions.

## Example Response Shape

### Candidate Summary

Game-oriented builder profile with evidence across public project work, visible product ownership, and technical execution. Strongest positioning comes from shipped or inspectable work rather than generic corporate summary language.

### Target Role Fit

- Strong fit for game production, technical design, indie product, or project-led builder roles
- Moderate fit for broader product or operations roles when evidence supports process ownership and outcome language

### Resume Draft

- Candidate summary focused on domain positioning
- Experience bullets rebuilt from visible evidence only
- Project and ownership signals preserved without inventing unsupported employment details
- One-page compression keeps the strongest signals first and trims repetition

### Weak Spots / Missing Inputs

- Exact dates and title progression may still need confirmation
- Metrics may be weak or unavailable in evidence
- Education claims should remain conservative unless directly supported

### Optional Tailoring Notes

- If a target job post is added later, keyword and responsibility alignment can be tightened without changing unsupported claims
